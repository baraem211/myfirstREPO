# 🌐 웹 개발 기초 용어 정리 가이드

이 문서는 웹과 앱 서비스의 구조를 이해하기 위한 핵심 용어들을 정리한 문서입니다.

---

## 1. 플랫폼 및 서비스 형태

### 📍 웹 (Web)
브라우저(크롬, 사파리 등)를 통해 접속하는 **망(Network)**이자 **웹사이트**를 통칭합니다.
* **특징:** URL 주소만 있으면 기기 종류에 상관없이 접속 가능.
* **비유:** 누구나 들어와서 읽을 수 있는 '온라인 잡지'.

### 📍 앱 (Application)
스마트폰이나 PC에 **직접 설치**해서 사용하는 소프트웨어입니다.
* **특징:** 카메라, GPS 등 하드웨어 기능을 밀접하게 사용하며 속도가 빠름.
* **비유:** 내 기기에 직접 설치해 쓰는 '전용 도구'.

### 📍 웹앱 (Web App)
겉모습은 앱 같지만, 실제로는 웹 기술로 만들어 브라우저에서 실행되는 형태입니다.
* **특징:** 설치가 필요 없으나 일반 웹보다 역동적인 인터페이스를 제공함.
* **비유:** 앱의 탈을 쓴 똑똑한 웹사이트.

---

## 2. 페이지 및 영역 구분

### 📍 랜딩페이지 (Landing Page)
사용자가 외부 광고나 링크를 클릭했을 때 **가장 먼저 도착하는 페이지**입니다.
* **목적:** 회원가입, 구매, 앱 다운로드 등 사용자의 특정 행동을 유도함.
* **비유:** 고객의 시선을 사로잡는 '온라인 전단지'.

### 📍 프론트엔드 (Frontend)
사용자가 눈으로 직접 보고 **상호작용하는 모든 부분**을 말합니다.
* **역할:** 디자인, 레이아웃, 버튼 반응, 애니메이션 구현.
* **기술:** HTML, CSS, JavaScript 등.
* **비유:** 식당의 인테리어와 손님을 맞이하는 '홀 서비스'.

### 📍 백엔드 (Backend)
눈에 보이지 않는 **서버 영역**에서 데이터를 처리하고 저장하는 영역입니다.
* **역할:** 데이터 관리, 회원 인증, 비즈니스 로직 처리.
* **기술:** Java, Python, Node.js, DB(MySQL, MongoDB 등).
* **비유:** 음식을 실제로 조리하고 재료를 관리하는 '주방'.

---

## 3. 프론트엔드 vs 백엔드 비교

| 구분 | 프론트엔드 (Frontend) | 백엔드 (Backend) |
| :--- | :--- | :--- |
| **관점** | 사용자 경험 (UX), UI 디자인 | 데이터 처리, 보안, 성능 |
| **핵심 도구** | React, Vue, HTML/CSS | Spring, Django, DB, API |
| **비유** | 메뉴판, 테이블, 조명 | 레시피, 조리 기구, 식재료 창고 |
